## Goal

`docs/plans/topic-generation-paths/04-checklist.md`의 **단계 1·2·3·4·5·6·8·9·11**을 한 번에 마쳐 e2e-rehearsal Scope A의 step 3·4(키워드 등록 + 자동 경로 주제 생성) 검증 가능 상태로 만든다.

**비범위(이 task 안 함)**: 단계 7(`TopicsScreen` 수동 버튼·직접 작성 진입점) — Scope A는 자동 경로만 검증. 단계 10(수동 E2E)은 사용자 본인 검증 영역.

라운드 1 + 사용자 일괄 수용(2026-04-26) 반영:
- Q1: 자동 경로는 키워드 추가 시 즉시 Claude 호출 (debounce/토글 없음).
- Q2: 자동 경로가 키워드 status를 자동 resolved로 바꾸지 않는다.
- Q3: 수동 "주제 생성" 버튼은 pending 0개여도 enabled, 누르면 안내 다이얼로그 표시.
- Q4: 옵션 풀 자동완성 UI는 본 task scope 제외 (#6 영역).
- Q5: 모든 경로의 default 태그 = `모든주제` 단독. Claude 응답에 태그 없음.

## User-visible behavior

- 키워드 화면(`KeywordScreen`)에서 키워드 추가 → 백그라운드에서 즉시 Claude `ClaudeTopicSuggester.suggest` 호출 → Topics DB에 1~N개 주제 추가 (N은 Claude가 결정). 태그는 모두 `모든주제` 단독. 사용자에게는 스낵바로 "주제 N개가 생성되었습니다" 알림.
- 키워드 화면 등록 폼에 태그 입력 컴포넌트(자유 텍스트 멀티 입력) 노출. `addKeyword(text, type, tags = listOf("자유태그1"))` 호출 가능.
- `TopicsScreen` 자체에는 큰 UI 변경 없음(단계 7 비범위). 기존 화면이 정상 컴파일되고 보이는 수준 유지.
- `DailyTopicWorker`와 그 스케줄 enqueue 제거 → 백그라운드에서 자동 생성이 시간 트리거로 일어나지 않음(사용자 행동 트리거만).

## Scope

### 단계 1 — `ClaudeTopicSuggester` 신설
- `app/src/main/java/com/dailynewsletter/service/ClaudeTopicSuggester.kt` 신설.
- README의 데이터 클래스 스펙 그대로:
  ```kotlin
  data class SuggestedTopic(
      val title: String,
      val priorityType: String,         // "direct" | "prerequisite" | "peripheral"
      val sourceKeywordIds: List<String>,
      val reason: String
  )
  ```
- `@Singleton` + `@Inject constructor(claudeApi, settingsRepository)`. side-effect 없음(저장은 ViewModel 책임).
- `suggest(pendingKeywords, pastTopicTitles): List<SuggestedTopic>` — Claude 메시지 호출 + JSON 응답 파싱.
- 프롬프트 핵심: 사용자가 등록한 pending 키워드 풀 + 이미 프린트된 주제 제목 목록을 입력하고, "이미 프린트된 주제와 중복되지 않는 주제 1~N개를 제안하라(N은 Claude 판단)"는 지시. **태그 필드 응답 X** (라운드 1).
- 프롬프트 상세는 `02-backend.md` §"Claude 프롬프트" 참조하되, 라운드 1 정합으로 `tags` 필드는 응답 스키마에 포함하지 않는다.

### 단계 5 — `KeywordViewModel` 자동 경로 orchestration
- `app/src/main/java/com/dailynewsletter/ui/keyword/KeywordViewModel.kt`:
  - `addKeyword(text, type, tags)` 호출 후, 반환된 `KeywordUiItem.id`를 모아 `ClaudeTopicSuggester.suggest`를 호출.
  - `suggest` 응답의 각 `SuggestedTopic`을 `TopicRepository.saveTopic(title, priorityType, sourceKeywordIds, tags = emptyList())`로 저장 (`ensureFreeTopicTag` invariant가 `[모든주제]` 보충).
  - 저장 후 사용자에게 스낵바로 "주제 N개가 생성되었습니다" 알림 (StateFlow로 UI 측에 전달).
  - 실패 시(Claude 호출 오류 등) 스낵바로 에러 메시지(스펙 §6 정책: 알림만, 자동 재시도 없음).
  - **Q1 채택**: 키워드 1건 추가마다 즉시 호출. debounce/토글 없음.
  - **Q2 채택**: 키워드 status 자동 resolved 변경 없음.
  - 동시성: `suggest` 결과 저장 중 또 다른 키워드가 추가돼도 별도 코루틴으로 처리 (구현 단순성을 위해 단일 큐로 sequential 처리해도 OK).
