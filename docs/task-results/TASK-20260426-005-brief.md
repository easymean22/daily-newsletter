## Scope

### 단계 1 — Notion 모델 확장
- `data/remote/notion/NotionApi.kt` / `NotionModels.kt`:
  - **단건 페이지 조회 엔드포인트**: `getPage(pageId): NotionPage` (없으면 추가). Retrofit `@GET("v1/pages/{pageId}")`.
  - **`Topics.Status`에 `consumed` 옵션 추가**: 기존 select 옵션 풀에 `"consumed"` 추가. (Notion select는 옵션을 동적으로 생성하므로 모델 변경은 불필요할 수 있음 — `NotionSelectValue("consumed")`로 직접 setProperty 가능. 단 schema 시드를 명시적으로 등록한다면 `NotionSetupService.kt`도 갱신.)
- 시드 갱신이 필요한 경우 `service/NotionSetupService.kt`의 Topics DB 생성 호출에 `consumed` 옵션을 추가. (단 Notion이 동적 생성을 허용한다면 스킵 가능.)

### 단계 2 — `NewsletterRepository` 신규 메서드
- `data/repository/NewsletterRepository.kt`:
  - `saveNewsletter(...)` 반환 타입 `Unit → String`(생성된 Notion page id) 변경.
  - `getNewsletter(id: String): NewsletterUiItem` 신규(단건 조회).
  - `updateNewsletterStatus(id: String, status: String)` 신규.
  - `findUnprintedByTagAndPages(tag: String, pageCount: Int): List<NewsletterUiItem>` 신규 (TASK-003에서 추가한 `findUnprintedNewsletterByTag`은 단건 first 반환 — 본 메서드는 리스트 반환에 `Page Count` 추가 필터 포함).
  - 기존 `printNewsletter(id)` 시그니처는 유지(IPP 호출까지 도달).
  - **N+1 쿼리 위험**(`getBlockChildren` per page) 인지: 본 task는 기존 `getNewsletters` 동작을 변경하지 않음. 최적화는 별도 task.

### 단계 3 — `TopicRepository` 신규 메서드
- `data/repository/TopicRepository.kt`:
  - `findPendingTopicsByTag(tag: String): List<TopicUiItem>` 신규 — Notion 쿼리 (`multi_select.contains` + `Status.equals("selected")` 등 pending 정의).
  - `markTopicsConsumed(ids: List<String>)` 신규 — 각 id의 `Status`를 `consumed`로 update.
  - **TASK-004가 본 파일에서 `TopicSelectionService` 의존 제거를 이미 했으므로**, 본 task는 그 결과 위에서 추가 메서드만 더한다. 기존 메서드(`saveTopic`, `getTodayTopics`, `getAllPastTopicTitles`, `ensureFreeTopicTag` 호출 게이트)는 그대로.

### 단계 4 — `NewsletterGenerationService.generateForSlot` 재작성
- `service/NewsletterGenerationService.kt`:
  - 기존 `generateAndSaveNewsletter()` 메서드는 deprecated 또는 삭제 후 `generateForSlot(tag: String, pageCount: Int): GeneratedNewsletter` 신설.
  - `GeneratedNewsletter(id, title, html, selectedTopicIds, tags)` 데이터 클래스(README 스펙).
  - 흐름:
    1. `topicRepository.findPendingTopicsByTag(tag)` 로 후보 Topic 조회.
    2. 후보 비면 예외 또는 nullable 반환 (사용자에게 알림).
    3. Claude 호출로 N개 주제 + HTML 생성. 프롬프트에 **"N ≤ pageCount × 2" 가이드** 한 줄 + "목표 글자수 = pageCount × 1800자" 가이드.
    4. 응답에서 `selectedTopicIds.size > pageCount * 2` 면 경고 로그(`Log.w`) 1줄 — 자르지 않음(html과 ID 불일치 회피).
    5. `newsletterRepository.saveNewsletter(title, html, selectedTopicIds, pageCount, tags)` 호출 → 반환된 id를 `GeneratedNewsletter.id`에.
    6. `topicRepository.markTopicsConsumed(selectedTopicIds)` 호출 (저장 직후).
  - Notion `richText.text.content`의 2000자 제한은 본 task scope 외 알려진 결함 — TODO 주석으로만 남기고 별도 task로 회수.
  - tags 매개변수: 라운드 1대로 emptyList()로 호출자에서 전달, `saveNewsletter` 내부는 그대로 전달(invariant는 Topics 한정이라 Newsletters는 별도 처리 X).
