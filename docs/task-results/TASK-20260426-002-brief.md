# Task Brief: Disconnect broken PrintWorker periodic enqueue

Task ID: TASK-20260426-002
Status: active
Owner: planner → implementer

## Goal

`WorkScheduler.schedulePrint(...)`이 `PrintWorker`를 periodic work로 enqueue하면서 input data를 채우지 않아, `PrintWorker.doWork()`가 항상 `Result.failure()` 로 끝나는 silent 실패 경로를 제거한다. MVP 1일 E2E 리허설 시점에 이 경로가 매일 깨진 상태로 fire되는 것을 막는다.

## User-visible behavior

- 기존: 사용자가 `Settings`에서 프린트 시각을 설정해도 자동 인쇄 시점에 아무 일도 일어나지 않음(notification도 없음). `Result.failure()`만 WorkManager 로그에 남음.
- 변경 후: 자동 주기 인쇄 경로가 비활성. 사용자는 **`NewsletterScreen`의 수동 `Print` 버튼**을 통해서만 인쇄. 이 경로는 `NewsletterRepository.printNewsletter(id)` → `PrintService` → IPP를 거치며 input data를 정상적으로 보유하므로 동작한다.
- spec §3 "사용자 행동 + 프린트 시점 트리거" 모델과 일치.

## Scope

- `WorkScheduler.scheduleAll()`에서 `schedulePrint(hour, minute)` 호출 라인 제거.
- `WorkScheduler.schedulePrint(...)` private 메서드 삭제.
- `WorkScheduler` 안에서 사용하지 않게 된 import는 함께 정리(예: `BackoffPolicy`가 다른 곳에서 안 쓰이면 제거).
- `WorkScheduler.cancelAll()`은 그대로 유지(이미 모든 work를 취소하므로 disconnect 후에도 안전).
- `PrintWorker` 클래스 자체는 **삭제하지 않는다** — `NewsletterRepository.printNewsletter(id)` 또는 향후 `PrintOrchestrator` 도입 시 재사용 후보. 단지 자동 enqueue만 끊는다.
